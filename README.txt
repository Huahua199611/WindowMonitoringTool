🪟 窗口监控工具 📸
📝 软件概述
这是一款专用于Windows系统的窗口监控工具，能够自动检测新打开的窗口并实时截取保存截图。 通过简洁直观的图形界面，帮助用户轻松记录和管理系统中出现的各类窗口信息。

✨ 主要功能 ✨
🔍 窗口监控：自动检测并记录系统中新打开的窗口
📸 自动截图：发现新窗口时立即截取并保存清晰截图
🖥️ 图形界面：简洁友好的操作界面，直观展示监控状态
📋 实时日志：详细记录所有监控活动和操作历史
⚡ 高效运行：采用多线程设计，确保监控过程不影响系统性能
🌐 中文支持：完全支持中文显示和中文路径
🛠️ 系统要求
操作系统：Windows 7/8/10/11 💻
依赖环境：
打包版本：无需额外安装依赖 🚀
源码运行：Python 3.6+，以及相关依赖库 🐍
🚀 快速开始
直接运行（推荐）
找到项目中的 窗口监控.exe 可执行文件 📄
双击运行即可启动程序 👆
在程序界面点击「开始监控」按钮开始使用 ▶️
从源码运行
确保已安装Python 3.6或更高版本 🐍
安装必要的依赖：

bash
pip install pywin32 pillow
运行主程序：

bash
python main.py
🎮 使用方法
启动程序：双击 窗口监控.exe 或通过命令行运行 ▶️
开始监控：点击界面上的「开始监控」按钮 🟢
停止监控：需要暂停监控时，点击「停止监控」按钮 🔴
查看日志：在程序界面下方的日志区域查看实时监控记录 📋
退出程序：点击「退出」按钮或关闭窗口 ❌
📁 截图保存
所有截图默认保存在程序同目录下的 pic 文件夹中 📂
截图文件命名格式：时间戳_窗口标题.png 📝
系统会自动创建 pic 文件夹（如果不存在） ✅
🖼️ 界面说明
标题栏：显示程序名称「窗口监控工具」 📌
操作区：包含「开始监控」、「停止监控」和「退出」三个功能按钮 ⚙️
日志区：实时显示程序运行状态和监控记录 📋
状态栏：显示当前程序状态（就绪/监控中/已停止监控） 📊
⚙️ 技术细节
开发语言：Python 🐍
核心库：
tkinter：构建图形用户界面 🎨
win32gui：获取和操作Windows窗口 🪟
PIL (Pillow)：处理和保存截图 📸
threading：实现多线程监控 ⚡
打包工具：Nuitka 📦
⚠️ 注意事项
程序需要足够的权限才能截取某些窗口的截图 🔑
为确保截图质量，监控到新窗口时会暂时将该窗口置于前台 🔄
大量新窗口同时打开时，可能会有短暂的性能影响 ⏱️
截图文件会占用一定磁盘空间，请定期清理不需要的截图 💾
🔧 常见问题
Q: 为什么某些窗口没有被监控到？ A: 程序只会监控可见且有标题的窗口，某些特殊窗口可能无法被检测。 ❓

Q: 截图保存在哪里？ A: 默认保存在程序同目录的 pic 文件夹中，可在代码中修改保存路径。 📂

Q: 程序运行时占用资源高吗？ A: 程序采用优化设计，正常监控状态下资源占用较低。 🔋

Q: 如何调整监控间隔？ A: 可以修改源代码中 WindowMonitor 类初始化时的 check_interval 参数。 ⏲️
